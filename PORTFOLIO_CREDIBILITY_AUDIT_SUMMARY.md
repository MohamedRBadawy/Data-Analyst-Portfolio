## THE PROBLEM WITH NARRATIVE ALONE

When an executive reads your portfolio, they're evaluating:

| What They See | What They Think |
|---|---|
| "128 hours → 1 minute" | "Sounds impressive, but how do I know?" |
| "41% defect reduction" | "Based on what measurement?" |
| "75% fewer errors" | "Did the team just stop counting?" |
| "Same-day onboarding" | "Proof?" |

**Your narratives are credible.** But credibility scales when you add proof artifacts.

---

## WHAT EXECUTIVES ACTUALLY WANT TO VERIFY

1. **For Automations:** "Does this system actually run that fast? Can I see the execution log?"
   - Current claim: "128 hours → 1 minute"
   - Proof needed: Execution log showing millisecond timestamp, record count, zero errors

2. **For Dashboards:** "Did the metric really improve? Is it sustained?"
   - Current claim: "41% defect reduction (1.68% → 0.99%)"
   - Proof needed: Trend graph showing week-by-week improvement over 6+ months

3. **For Processes:** "How did you actually measure the impact?"
   - Current claim: "Reduced from 2-3 days to same-day"
   - Proof needed: Before/after timeline with timestamped examples

4. **For Any Claim:** "Is this real or exaggerated?"
   - Current exposure: Founders' self-reported claims
   - Proof needed: Third-party verifiable artifacts (logs, dashboards, trend data)

---

## THE 3-ARTIFACT FORMULA

Every project should have **exactly 3 proof artifacts**:

### **For Automations** (Projects 1, 4, 5, 7, 8):
1. **Execution log** → Proves speed (milliseconds, not minutes)
2. **Input/Output samples** → Proves transformation quality
3. **Process flow diagram** → Proves methodology

### **For Dashboards** (Projects 2, 3, 6):
1. **Before/After dashboard screenshots** → Proves tool exists and transformed
2. **Trend graph** → Proves metric moved (week-by-week, not snapshot)
3. **Drill-down capability demo** → Proves analytical power

### **For All Projects**:
- Anonymize client/employee data (redact names, use labels A/B/C)
- Show real execution logs/screenshots (not mockups)
- Include month-over-month trends (shows sustainability)

## WHAT TO ABSOLUTELY AVOID

| ❌ DO NOT SHARE | Why It Kills Credibility |

| Live system links | Exposes client infrastructure |
| Employee names/data | Privacy violation |
| Full source code | Invites nitpicking of implementation |
| Client names | Confidentiality breach |
| Unverified metrics | Just more claims to verify |
| Vague before-state | Looks exaggerated |
| Marketing language | Undermines proof-focused strategy |
